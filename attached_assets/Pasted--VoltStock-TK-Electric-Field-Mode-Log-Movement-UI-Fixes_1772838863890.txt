[VoltStock / TK Electric - Field Mode Log Movement UI Fixes (Receive/Return vs Issue/Transfer)]

Goal:
Fix Field Mode log movement UX to match intended flows and eliminate confusing options. Keep current design style (green/clean), but improve readability and navigation. Must be fully functional end-to-end (DB persistence).

=====================================================
A) Split Movement Type options by screen (VERY IMPORTANT)
=====================================================
1) In FIELD MODE "Receive/Return" screen:
- Movement Type dropdown MUST ONLY show:
  - Receive
  - Return
- REMOVE Issue, Transfer from this screen completely.

2) In FIELD MODE "Issue/Transfer" screen (rename from Issue/Ship):
- Rename UI label everywhere:
  - "Issue/Ship" -> "Issue/Transfer"

  - Issue
  - Transfer

(Implementation note)
- Do NOT just hide in UI while API accepts others.

- Ensure list is not clipped by parent containers (z-index / overflow hidden issues)
- Keep the Item input height normal, but expand the dropdown list area.
