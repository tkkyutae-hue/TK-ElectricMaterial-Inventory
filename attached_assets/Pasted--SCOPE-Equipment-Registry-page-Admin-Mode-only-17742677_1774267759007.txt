FIX 1 — EQ # is optional, not required

  Current: saving without an EQ # is blocked or
           produces an error.
  Fix:     EQ # is optional at save time.

  - If admin leaves EQ # blank, auto-assign a
    sequential fallback (e.g. EQ-001, EQ-002…)
    and display it in the saved row.
  - Admin can overwrite the fallback later via edit.
  - Do NOT block save when EQ # is empty.


FIX 2 — Edit mode must use the same dropdowns as Add mode

  Current: clicking ✏ on a saved row switches the
           cells to plain <input> text fields.
           Name / Size / Brand lose their dropdowns.

  Fix:     edit mode must render the same creatable
           dropdown components as the add-row, not
           plain inputs.

  Behavior in edit mode:
  - NAME cell  → creatable dropdown (type selector),
                 pre-selected to current value
  - SIZE cell  → creatable dropdown, options based
                 on current type, pre-selected
  - BRAND cell → creatable dropdown, options based
                 on current type, pre-selected
  - EQ #       → editable text input (same as add row)
  - LOCATION   → text input + datalist (same as add row)
  - PROJECT / STATUS / LAST UPDATED → read-only,
                 no change (live fields)
  - Save / Cancel buttons appear in the actions cell

  Edit mode must feel identical to add mode in terms
  of input components. Only the row state differs
  (editing existing vs creating new).


FIX 3 — Creatable dropdowns for Type, Size, Brand

  All three dropdown fields (TYPE, SIZE, BRAND) must
  support adding and deleting options inline.
  This applies to BOTH add mode and edit mode.

  Panel layout when dropdown is open:
  ┌──────────────────────────────────┐
  │ [ Type to add new…       ] [ + ] │  ← add row at top
  │ ──────────────────────────────── │
  │  Air Compressor             [✕]  │
  │  Generator                  [✕]  │
  │  Forklift           ✓       [✕]  │  ← selected item
  │  Scissor Lift               [✕]  │
  │  …                               │
  └──────────────────────────────────┘

  ADD behavior:
  - Admin types in the input and presses [ + ] or Enter
  - New option appended to the bottom of the list
  - Immediately selected after adding

  DELETE behavior:
  - Each list item shows [ ✕ ] on hover
  - Clicking removes that option from the list
  - If deleted item was selected, clear selection

  TYPE → SIZE / BRAND linkage:
  - Selecting a type still auto-populates size and
    brand dropdowns with their default options
  - Custom-added options in size/brand are preserved
    for the current session

  Empty state:
  - If all options deleted, show:
    "No options — type above to add"

  Panel style:
  - Positioned below trigger, z-index clears table rows
  - max-height ~200px with scroll
  - border-radius 8–9px, subtle box-shadow
  - [ + ] button: small, square, green (#1a472a)
  - [ ✕ ] button: 18×18px, red tint (#fee2e2 / #f87171),
    visible on hover only
  - Selected item: ✓ checkmark left of label,
    light green row highlight
