Refine the existing VoltStock Replit project by implementing the Inventory UI/UX based on the current design direction.

Do not restart the app.
Do not replace the entire project.
Apply these changes directly to the existing codebase and preserve the current clean modern VoltStock design language.

## Main Goal

Update the Inventory experience so it matches this UI direction:

* a modern inventory page with category image cards at the top
* a searchable and filterable inventory table below
* visual category-based browsing
* electrical-industry-specific inventory organization
* SKU shown before Size in the inventory UI
* grouped category inventory views for easier stock comparison by size

The result must feel like a real internal inventory platform for a U.S. electrical contractor.

---

## 1. Inventory Page Layout

Redesign the Inventory page into this structure:

### Top section

* page title: Inventory
* short subtitle describing inventory management
* primary action button: Add New Item

### Filter / search row

Include:

* search input with placeholder like “Search by SKU or Name...”
* status filter
* location filter
* supplier filter if practical

Make the filters clean, compact, and aligned horizontally on desktop.
Preserve responsive behavior for smaller screens.

### Category card grid

Below the filters, add a responsive grid of category cards.

Each category card must include:

* representative image
* category name
* total SKU count
* total quantity summary
* low-stock count indicator

Use the existing electrical categories:

* CABLE TRAY
* CONDUIT / FITTINGS
* CONDUIT SUPPORTS & STRUT SYSTEM
* CABLE / WIRE
* DEVICES
* FASTENERS & HARDWARE
* BOXES & COVER SYSTEM
* DISTRIBUTION & POWER SYSTEM
* GROUNDING & TERMINATION

The cards must look modern and visually strong, but still fit a business dashboard.
Use soft radius, subtle shadow, clean spacing, readable text, and image thumbnails.

### Inventory table section

Keep the inventory table below the category cards.

Required visible columns:

* SKU
* Item Name
* Size
* Category
* Quantity
* Unit
* Location
* Status
* Reorder Point

Important:
Display SKU before Size in the table.
