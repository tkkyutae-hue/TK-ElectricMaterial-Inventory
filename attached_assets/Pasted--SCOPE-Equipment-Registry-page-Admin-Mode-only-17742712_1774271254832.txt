CHANGE 2 — EQ # auto-generation
──────────────────────────────────
EQ # is always optional. Never block save if empty.

If admin leaves EQ # blank:
  Auto-generate from: TYPE + SIZE + BRAND
  Format: {TYPE_ABBR}-{SIZE_SHORT}-{BRAND_SHORT}

  Type abbreviations:
    Air Compressor   → AC
    Generator        → GEN
    Forklift         → FK
    Scissor Lift     → SL
    Boom Lift        → BL
    Conduit Bender   → CB
    Wire Puller      → WP
    Drill            → DR
    Grinder          → GR
    Vacuum           → VAC
    Power Distribution → PD
    Cable Puller     → CP
    Other            → EQ

  Size: strip spaces, keep alphanumeric, uppercase, max 5 chars
    e.g. "5,000 lbs" → "5000L", "32 ft" → "32FT"

  Brand: first 3 chars uppercase
    e.g. "Genie" → "GEN", "Honda" → "HON"

  Example results:
    Scissor Lift + 32 ft + Genie   → SL-32FT-GEN
    Generator + 5,500W + Honda     → GEN-5500W-HON
    Boom Lift + 45 ft + JLG        → BL-45FT-JLG
    Forklift + 5,000 lbs + Toyota  → FK-5000L-TOY

  Show auto-preview below the EQ # input in real time:
    Small green text, 9px
    Checkmark icon + "Auto: {generated_value}"
    Only visible when EQ # input is empty AND
    at least one of type/size/brand is selected

  On save with blank EQ #: use the auto-generated value.
  If all three fields empty: fall back to sequential
    EQ-001, EQ-002, etc.
