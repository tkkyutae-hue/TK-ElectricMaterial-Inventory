현재 프로젝트의 Excel export 기능을, 내가 수정해서 업로드한 최종 엑셀 디자인 파일 기준으로 반영해줘.

중요:
- 최소 수정으로 구현
- 기존 export 기능이 이미 동작하고 있으니, 새로 갈아엎지 말고 현재 export 로직을 확장하는 방식으로 수정
- 다른 페이지/기능은 건드리지 마
- 실제 프로젝트 데이터 기준으로 구현
- 하드코딩 샘플 데이터 금지

수정 범위:
1. client/src/pages/Inventory.tsx
2. server/routes.ts
3. server/storage.ts
4. export 관련 유틸 파일이 따로 있으면 그 파일만 추가 수정
5. shared/schema.ts 는 참고만

목표:
- 현재 전체 자재 export 기능을
- 내가 수정한 최종 엑셀 디자인 스타일에 맞춰서 내보내기
- 카테고리별 sheet 구조는 유지
- 기존 단순 테이블 dump 형태가 아니라, 최종 템플릿처럼 정리된 구조로 export

파일명 규칙:
- 내보내기 파일명은 아래 형식으로 생성
- GA WAREHOUSE MATERIAL STATUS-YEAR-MONTH(NUMBER).xlsx
- 예:
  - 같은 달 첫 파일: GA WAREHOUSE MATERIAL STATUS-2026-04(1).xlsx
  - 같은 달 두 번째 파일: GA WAREHOUSE MATERIAL STATUS-2026-04(2).xlsx

중요 파일명 규칙:
1. YEAR는 4자리
2. MONTH는 2자리
3. NUMBER는 같은 달 기준 순번
4. 브라우저/서버가 이전 파일 존재 여부를 직접 알 수 없으면, 우선 기본값 (1)로 처리해도 됨
5. 추후 번호 로직 확장 가능하게 파일명 생성 함수를 분리해줘

디자인/구조 요구사항:
내가 수정한 최종 엑셀 파일 스타일을 기준으로 반영해줘.

1. 컬럼 구성은 기존 많은 컬럼이 아니라 아래 8개만 사용:
- Material Name
- Size
- Family
- Type
- Quantity
- Unit
- Status
- Last Updated

2. 각 카테고리 sheet 내부 구조:
- 최상단: 헤더 행
- 그 아래 데이터는 단순 나열이 아니라 그룹 구조로 정리
- 대분류 행
- 중분류 행
- 실제 자재 행
이 순서로 정리

즉 한 sheet 안에서 구조는 이런 느낌이어야 함:
- Header row
- Group row level 1 (대분류)
- Group row level 2 (중분류)
- Item rows

3. 그룹핑 규칙:
- 현재 프로젝트의 실제 데이터 구조를 보고
- 가능한 한 Family / Type / Material Name 관련 구조를 활용해서
- 최종 템플릿과 최대한 비슷한 그룹 구조를 만들어줘
- 무리한 추측은 하지 말고, 현재 데이터 기준에서 가장 자연스럽게 정리
- 대분류/중분류 텍스트는 기존 데이터의 family/type/name 구조를 활용

4. 스타일 요구사항:
최종 업로드된 디자인 파일 느낌을 최대한 맞춰줘.

- 헤더 행:
  - 진한 다크 배경
  - 흰 글씨
  - bold
- 대분류 행:
  - 진회색 강조
  - bold
- 중분류 행:
  - 연한 베이지/밝은 강조색
  - bold 또는 반강조
- 실제 데이터 행:
  - 일반 흰 배경
- 상태값 스타일:
  - In Stock: 녹색 계열 강조
  - Out of Stock: 빨간 계열 강조
- Quantity가 0이면 빨간 계열 강조
- Last Updated는 읽기 쉬운 날짜 형식으로 출력

5. 시트별 공통 포맷:
- 컬럼 너비는 최종 템플릿처럼 읽기 좋게 조정
- 첫 행 헤더 고정(freeze panes)
- freeze pane은 A2 기준으로 고정
- 자동 필터는 헤더 기준으로 적용 가능하면 적용
- 너무 긴 값은 실무에서 읽기 좋은 수준으로 정리
- 시트명은 기존처럼 카테고리명 사용하되 Excel 금지문자 처리 유지
  - "/" 는 "&" 로 치환 유지
  - 길이 제한 31자 처리 유지

6. 데이터 규칙:
- 전체 inventory export 유지
- 현재 화면 pagination/filter와 무관하게 전체 active item 기준
- null/undefined는 빈칸
- quantity는 숫자형 유지
- status는 현재 프로젝트 로직 기준값 사용
- last updated 값이 있으면 표시, 없으면 빈칸
- 사람이 읽기 쉬운 값으로 넣기

중요 구현 방향:
1. server/storage.ts
- 전체 active item을 export용으로 가져오는 기존 로직 재사용
- family, type, material name, quantity, unit, status, updatedAt 등 export에 필요한 값 정리

2. server/routes.ts
- 기존 xlsx export API를 유지하면서
- workbook 생성 구조만 최종 디자인 기준으로 업그레이드
- 시트별 헤더/그룹/데이터 스타일 반영
- 파일명 규칙을 새 형식으로 적용

3. client/src/pages/Inventory.tsx
- 기존 Export Excel 버튼 유지
- 버튼 클릭 시 새 파일명이 다운로드되도록만 연결 확인
- 기존 다운로드 동작 깨지지 않게 유지

절대 하지 말 것:
- 다른 페이지 리팩토링
- inventory 화면 구조 변경
- export와 관계없는 코드 수정
- 샘플 데이터 하드코딩
- 디자인 파일을 단순 수동 참고만 하고 무시하는 것

작업 순서:
1. 현재 export 로직 확인
2. 내가 수정한 최종 엑셀 파일의 구조/스타일 참고
3. 기존 export workbook 생성 부분만 최소 수정
4. 그룹 구조 + 스타일 + 파일명 반영
5. 기존 다운로드 동작 유지 확인
