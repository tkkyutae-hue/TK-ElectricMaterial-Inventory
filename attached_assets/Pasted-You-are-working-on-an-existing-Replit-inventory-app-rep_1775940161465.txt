TASK GOAL

BUSINESS INTENT
Undo should be available for "just performed" actions when the action can be safely rolled back.
This includes:
- cart item removal
- cart clear
- request submit
- request update
- request cancel
- request status changes where safe
- request completion, even when a real movement/transaction was created, as long as the rollback is still safe

IMPORTANT SAFETY RULE
Do NOT make Undo universal without limits.
Undo must be allowed only when:
- the action just happened
- the action came from the current user/session flow
- the rollback is well-defined
- no conflicting downstream action has already happened

==================================================

- show a success toast with Undo

Undo rules for completion:
1. Undo is allowed only for the just-completed request from the current action flow
2. Undo is allowed only if the completion created exactly the expected movement for this request
3. Undo is allowed only if no conflicting downstream action has occurred after completion
4. If safe, Undo should:
   - reverse/remove/rollback the created movement using the repo’s safest existing movement undo/delete pattern
   - restore the request status from `completed` back to `ready`

   - restore inventory state consistently
5. If undo is no longer safe, show a clear message and block it instead of forcing a dangerous rollback

Preferred state restoration:
- previous status before completion = `ready`
- after undo, request returns to `ready`

Important:
- Do NOT create a second compensating transaction unless that is the repo’s existing safe pattern
- Prefer the repo’s current safe movement rollback/delete mechanism if one already exists
- Prevent duplicate undo

==================================================

Before allowing completion undo, check things like:
- request is still in completed state
- request has the linked fulfilled movement id/reference

- no later conflicting request state change already happened
- no other obvious dependent operation makes rollback unsafe

If any safety check fails:
- do NOT undo
- show a clear message like:
  - "Undo is no longer available for this completed request"
