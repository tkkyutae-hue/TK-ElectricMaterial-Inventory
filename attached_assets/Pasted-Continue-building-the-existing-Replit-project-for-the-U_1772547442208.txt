## Inventory Status Logic

This logic must be implemented centrally, not copied randomly in many places.

### Rules

* In Stock = quantity_on_hand > reorder_point
* Low Stock = quantity_on_hand <= reorder_point and quantity_on_hand > 0
* Out of Stock = quantity_on_hand = 0
* Ordered = status_override = 'ORDERED' or similar explicit flag

Create a clean helper/service for status calculation.

Also expose:

* computed status label
* status color
* whether reorder is recommended
