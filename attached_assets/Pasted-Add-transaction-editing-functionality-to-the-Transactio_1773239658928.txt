════════════════════════════════════════

States:
  0 rows selected:
    Text: "Select rows to edit or delete" (muted color)

    Text: "1 record selected"

    Edit button: disabled (opacity 0.35) — editing multiple not supported

Button styles:
  Edit:   background rgba(45,219,111,0.10), border var(--g-bd), color var(--green), glow shadow

════════════════════════════════════════
4. SAVING BEHAVIOR
════════════════════════════════════════
On Save:
  a. Update the transaction record in the database with all edited values
  b. Auto-set: edited_by = current user, edited_at = current timestamp
  c. Do NOT modify: created_at, created_by, transaction_id
  d. Close the drawer
  e. Immediately reflect changes in the table row (no page reload)
  f. Show success toast (see below)

DATA FIELDS TO STORE (add to transactions schema if not present):
  edited_by   — string, user who last edited (null if never edited)
  edited_at   — timestamp, when last edited (null if never edited)
  edit_history — optional JSON array: [{edited_by, edited_at, changed_fields: {field: {old, new}}}]
