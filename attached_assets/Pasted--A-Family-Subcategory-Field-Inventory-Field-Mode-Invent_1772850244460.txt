[A안 - 전체 카테고리 고정밀 자동 Family(Subcategory) 재분류 + Field Inventory 필터 정상화]

목표:
- Field Mode > Inventory에서 Category 클릭 후 Family(Subcategory) 필터가 의미 있게 여러 개 뜨도록,
  DB의 items.subcategory를 SKU 패턴 + name 키워드 기반으로 고정밀 자동 분류하여 일괄 업데이트한다.
- “카테고리 → Family → Size → Status” 필터가 전 카테고리에서 정상 작동해야 한다.
- UI만 바꾸지 말고 DB 데이터까지 실제로 수정한다.
- 분류는 “중복 실행해도 같은 결과”가 나오게(멱등성) 구현한다.

========================================================
0) 전제 (하드코딩 금지)
========================================================
- categories 테이블에서 name으로 category_id를 조회해서 사용 (id 하드코딩 금지)
- items 테이블에서 subcategory 컬럼명이 다르면 실제 컬럼명으로 매핑
- 분류 결과는 items.subcategory에 저장

필수 출력(완료 후):
- 카테고리별 subcategory count 요약
- 특히 CABLE TRAY / CONDUIT / CABLE-WIRE / POWER / BOXES는 상세 출력

========================================================
1) 분류 엔진 구현 방식 (정확도 향상)
========================================================
각 카테고리는 아래 순서로 subcategory를 결정한다:

(1) SKU prefix 규칙 (가장 우선, 신뢰도 최고)
(2) name 키워드 규칙 (정규식/ILIKE)
(3) 폴백: "Other"

우선순위는 카테고리별로 명확히 둔다 (예: Cover가 Tray보다 먼저).

분류는 admin-only 스크립트 또는 endpoint로 구현:
- 실행 시 해당 카테고리 items만 UPDATE
- 변경된 row 수 및 서브카테고리별 집계 로그 출력
- 재실행해도 결과 동일 (멱등)

========================================================
2) CABLE TRAY (CABLE TRAY)
========================================================
목표 subcategory:
- Tray
- Covers
- Fittings
- Connectors
- Supports
- Accessories
- Other

(1) SKU 기반 (가능한 범위에서 적용)
- CT-CVR-*  -> Covers
- CT-COV-*  -> Covers
- CT-STR-*  -> Tray
- CT-TRAY-* -> Tray
- CT-TEE-*  -> Fittings
- CT-ELB-*  -> Fittings
- CT-CRS-*  -> Fittings
- CT-WYE-*  -> Fittings
- CT-RED-*  -> Fittings
- CT-VIN-* / CT-VOU-* -> Fittings
- CT-CON-* / CT-BC-* / CT-SPL-* / CT-JNT-* / CT-COU-* -> Connectors
- CT-HNG-* / CT-BKT-* / CT-CLP-* / CT-SUP-* -> Supports
- CT-END-* / CT-DIV-* / CT-DRO-* / CT-BND-* -> Accessories

(2) name 기반 우선순위: Covers > Fittings > Connectors > Supports > Accessories > Tray > Other
- Covers: cover|lid
- Fittings: elbow|tee|cross|wye|reducer|vertical|horizontal|inside|outside
- Connectors: splice|connector|coupler|coupling|joint
- Supports: hanger|support|bracket|trapeze|hold down|clamp|strap
- Accessories: end cap|divider|drop out|bond|bonding|jumper
- Tray: tray|ladder|wireway

========================================================
3) CONDUIT / FITTINGS (CONDUIT / FITTINGS)
========================================================
목표 subcategory:
- EMT Conduit
- RMC/IMC Conduit
- PVC Conduit
- Flex Conduit (LFMC/Liquidtight/ENT)
- Conduit Bodies (LB/LL/LR)
- Fittings (connectors/couplings/adapters/bushings/locknuts)
- Supports (straps/clamps)
- Other

우선순위: Flex > Conduit Bodies > Supports > Fittings > EMT > RMC/IMC > PVC > Other

(1) SKU 기반 예시
- EMT-* -> EMT Conduit
- RMC-* / IMC-* -> RMC/IMC Conduit
- PVC-* -> PVC Conduit
- LQT-* / LFMC-* / FLEX-* -> Flex Conduit
- *-LB-* / *-LL-* / *-LR-* -> Conduit Bodies

(2) name 기반
- Flex: liquidtight|lfmc|flex|ent
- Bodies: conduit body| LB | LL | LR
- Supports: strap|clamp|hanger
- Fittings: coupling|connector|adapter|bushing|locknut|compression|set screw|elbow
- EMT/RMC/IMC/PVC: emt|rmc|imc|pvc|sch 40|sch 80

========================================================
12) 실행 후 Field Inventory 검증
========================================================
- 각 카테고리 클릭 시 Family 드롭다운에 최소 3개 이상(가능한 경우) 의미 있는 옵션이 떠야 한다.
- Cable Tray에서 Tray만 뜨면 실패 (Covers/Fittings/Connectors/Supports 등 떠야 함)
- Conduit에서 EMT/RMC/PVC/Fittings/Supports 등이 떠야 함
- Family 선택 → Size 필터 옵션이 해당 scope로 좁혀져야 함
- Status 필터도 정상 동작

========================================================
13) 결과 리포트 필수
========================================================
- categories별 subcategory count top 10 출력
- CABLE TRAY, CONDUIT, POWER, BOXES는 subcategory별 전체 count 출력
- 변경된 row 수 출력
