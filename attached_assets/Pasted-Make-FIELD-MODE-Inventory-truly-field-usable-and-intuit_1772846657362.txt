Make FIELD MODE → Inventory truly field-usable and intuitive.
Do NOT redesign the whole app.
Keep current visual language (TK green, clean cards).
Everything must work end-to-end with DB + API (no UI-only placeholders).

====================================================
GOAL (FIELD USABILITY)
====================================================
Current Field Inventory is “category image → filter chip → end”.
Field crew needs fast drill-down and quick lookup on phone/tablet:
- Category → Subcategory(Family) → Size → Status
- Search always works (partial match)
- 10/page pagination
- Fast “tap-tap” workflow, minimal typing

====================================================
A) FIELD INVENTORY: DRILL-DOWN FILTER FLOW
====================================================
Implement a clear drill-down flow with 4 filters in one row:
1) Category (tap image card)  ✅ already exists but stops
2) Family/Subcategory dropdown (dependent on category)
3) Size dropdown (dependent on category + optional family)
4) Status dropdown (All / In Stock / Low / Out)

Rules:
- Changing Category resets Family + Size to All, page=1
- Changing Family resets Size to All, page=1
- Status can remain when Category changes (ok) but page=1
- Search text persists unless user clears it

UI behavior:
- After Category is selected, auto-focus the Family dropdown (or highlight it) so user knows next step.
- Show active filters as chips above the table: Category, Family, Size, Status, Search
  - Each chip has an X to clear that specific filter.
- Add one “Clear All” button next to chips.

====================================================
B) SEARCH MUST BE FIELD-FRIENDLY (PARTIAL MATCH)
====================================================
Fix search so it matches words contained anywhere (case-insensitive):
- Query should match if ALL tokens are found in name/SKU/size/family/category text.
Example:
- Searching `3/4 conduit` must match `3/4" EMT Conduit`.
Implement tokenization: split by spaces, trim, ignore empty tokens.
Server-side filtering (SQL ILIKE with AND across tokens).

====================================================
C) TABLE: ADD PHOTO + CATEGORY + LOCATION (FIELD VIEW)
====================================================
Field table must show at a glance what it is:
Columns (fixed order):
- SKU
- PHOTO (small thumbnail)
- ITEM (name + size under name if available)
- CATEGORY
- LOCATION
- QTY
- STATUS badge

Requirements:
- PHOTO column uses item image_url if available; fallback to placeholder.
- STATUS badge vertically centered in cell.
- Column widths consistent, no wrapping that breaks readability.
- Keep 10/page selector. Options: 10 / 15 / 20 / 25 (Field needs small pages).

====================================================
D) CATEGORY CARDS: FIX “SQUISHED” IMAGES + BETTER TAPS
====================================================
Category image cards currently look squished.
Fix card image styling:
- Maintain correct aspect ratio (use object-cover and a consistent container height).
- Make cards taller on field inventory page (so images are readable).
- Entire card is clickable; show selected state with border + subtle glow.

====================================================
E) SUBCATEGORY (FAMILY) VISIBILITY (MORE INTUITIVE)
====================================================

- Provide a secondary “Family quick bar” right under category cards:
  - top 6 most common families (based on item count) as pill chips

====================================================
F) SIZE FILTER UX
====================================================
Size dropdown should be auto-populated based on current scope:
- If Category selected: sizes from that category
- If Category + Family selected: sizes from that family
- Respect search text if present (sizes from filtered result)
Keep size display exactly as stored; no conversions.

Also add a quick sort toggle:
- “Size ↑/↓” (smallest-to-largest and reverse)
If size is not numeric (like “Standard”), sort those after numeric sizes.

====================================================
G) STATUS FILTER RULES (CONSISTENT)
====================================================
Status logic must match the app’s existing badges:
- Out of Stock: qty_on_hand <= 0
- Low Stock: qty_on_hand > 0 AND qty_on_hand <= (reorder_point or min_stock_level)
- In Stock: qty_on_hand > threshold
Use the same threshold field used elsewhere in the app.
